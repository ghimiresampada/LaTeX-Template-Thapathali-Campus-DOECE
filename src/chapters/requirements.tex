\chapter{REQUIREMENTS ANALYSIS}
\label{chap:requirements}

The development of a distributed, tile-based \gls{hpd} system requires a multi-faceted analysis of hardware constraints, software dependencies, and network reliability. This section outlines the essential requirements to ensure the feasibility of the project.

\section{Hardware Requirements}
Since the project targets the ESP32-S3 platform, the hardware selection is dictated by the need for \gls{ai} acceleration and power efficiency.

\begin{itemize}
    \item \textbf{Camera Module:} A high-resolution camera module(OV2640) capable of capturing images at higher resolution than traditional image processing systems, which will then be partitioned.
    \item \textbf{Input Node:} A master node responsible for taking images from the image sensor; partitioning them into multiple tiles and distributing them to edge nodes.
    \item \textbf{Edge Computing Nodes:} Multiple ESP32-S3 \glspl{mcu} are required. These must support the $Xtensa® LX7$ vector instructions to handle the tile-level \gls{cnn} inference.
    \item \textbf{Aggregator Node:} A  simple ESP32 responsible for collecting feature vectors and executing the final classification layer.
    \item \textbf{Communication Interface:} A low-latency communication system for inter-node communication \glspl{espnow}.
\end{itemize}

\section{Software \& Framework Requirements}
To achieve ultra-low bit quantization and distributed processing, the following software stack is required:

\begin{itemize}
    \item \textbf{Development Environment:} Arduino IDE using C/C++.
    
    \item \textbf{AI Inference Library:} \gls{esp-dl} to utilize high-performance kernels for quantized 8-bit or 16-bit operations.
    
    \item \textbf{Model Optimization:} \gls{tfml} for initial model training and conversion, followed by \gls{qat}.
    
    \item \textbf{Tiling Logic:} A custom C++ pre-processing module to handle spatial partitioning and memory-safe buffer management for individual tiles.
\end{itemize}


\section{Non-Functional Requirements}
\begin{itemize}
    \item \textbf{Latency:} The total time must be low enough for real-time processing.
    
    \item \textbf{Scalability:} The system should support a variable number of nodes depending on the resolution and required \gls{fps}.
    
    \item \textbf{Robustness:} In the event of a node failure, the system should gracefully degrade, relying on the remaining tiles to provide partial occupancy data.
\end{itemize}
