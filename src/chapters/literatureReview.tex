\chapter{LITERATURE REVIEW}
% The trajectory of smart building sensor technology is increasingly moving toward complex computer vision solutions to overcome the inherent failures of traditional sensing systems. The evolution from basic detection technologies like \gls{pir} and \gls{ble} is driven by the demand for superior accuracy and the critical need to differentiate between simple movement and actual physiological presence. Traditional \gls{pir} sensors are highly sensitive to thermal movement but fundamentally lack the ability to confirm the presence of a stationary individual, resulting in unreliable occupancy data for HVAC or smart lighting. While hybrid systems combining \gls{pir} and Radar have emerged to reduce false alerts, they still struggle with high-fidelity requirements \cite{Ahmad2021}. High-fidelity presence detection requires the capacity to sense subtle physiological indicators such as posture shifts or minute breathing patterns. Literature benchmarks indicate that Passive Infrared (\gls{pir}) sensors have a stationary detection reliability of approximately 0.0\%, making them ineffective for modern energy management needs when compared to PC-based models that achieve an 85.0\% Mean Average Precision (mAP). This project’s computer vision approach is designed to replace these specialized sensors by analyzing visual light patterns for high-frequency components that correspond to human activity, resolving the stationary detection issue by maintaining a presence signal even when movement is minimal.

% To bridge the gap between complex deep learning models and the extreme hardware limitations of edge microcontrollers, quantization is employed as a critical optimization strategy. By reducing the precision of the model's numerical values, it becomes possible to fit sophisticated human detection algorithms into a very small memory footprint \cite{nagel2021white, esp32s3_datasheet}. The industry standard involves converting 32-bit floating-point (FP32) weights into 8-bit integers (INT8), providing a four-fold reduction in memory. For models like MobileNetV2, Post-Training Quantization (PTQ) has been shown to retain up to 99.7\% of baseline accuracy. While extreme 4-bit \gls{int4} quantization allows for even smaller models, it often introduces a loss in precision that requires recovery techniques like layer-wise equalization. Modern approaches, such as those used in Q-CNN, prioritize response fidelity rather than just weight error, achieving up to 20x model compression with less than a 1\% drop in performance \cite{wu2016qcnn}. For this project, quantization is a necessity to meet the strict 512 \gls{kb} \gls{sram} ceiling of the ESP32-S3, as standard single-node edge solutions typically struggle with throughputs of only 1.0–1.5 FPS and inference latencies ranging from 800 ms to 1200 ms.

% To deploy these models effectively, the literature emphasizes optimizing both the arrangement of layers and the nature of the weights themselves. The SqueezeNet architecture achieves high accuracy with minimal parameters through the "Fire Module," which uses $1 \times 1$ "Squeeze" filters to reduce channels before they reach expensive $3 \times 3$ "Expand" filters \cite{iandola2016squeezenet}. Complementing this, Local Binary Convolutional Neural Networks (LBCNN) represent a paradigm shift toward fixed stochastic projections. Rooted in \gls{lbp}, LBCNN replaces standard learnable filters with fixed, sparse, binary kernels. In an LBC layer, the response is calculated as a learnable linear combination of these pre-defined binary filters, providing an immediate 9x reduction in parameters for $3 \times 3$ kernels, scaling up to 169x in deeper architectures. Because binary kernels are compatible with bitwise operations, LBCNN facilitates lower latency and reduced power consumption, allowing deep models to operate within the 512 KB SRAM limits of devices like the ESP32-S3 \cite{juefei2017lbcnn}.

% To avoid Out of Memory (OOM) crashes and overheating, which occur when a single chip handles massive 4K images, the literature suggests a distributed approach. While "Model Partitioning" exists, "Data Partitioning" or Tiling—which this project uses—is preferred for high-resolution data \cite{li2020pytorch}. This horizontal split involves cutting the image into a grid of independent tiles, allowing each node to run the full model in parallel and preserving the fine details usually lost when an image is shrunk. However, distributed systems must mitigate "Communication Overhead"; if the transfer time ($T_{comm}$) exceeds the computation time ($T_{comp}$), the system slows down \cite{jeong2021}. To solve this, the project utilizes the ESP-NOW protocol, offering low-latency peer-to-peer communication (1.115 \gls{ms}) and handling limited bandwidth (500–771 KB/s) by transmitting small result packets rather than raw images. This hierarchy includes an Input Node (ESP32-CAM) for tile partitioning, Edge Computing Nodes (ESP32-S3) for parallel inference, and an Aggregator Node to reconcile detections.

% Current research highlights two major gaps that this project aims to bridge. First, the Information Loss Gap occurs because most systems resize 4K streams to fit tiny models, acting as a blur filter that erases subtle human features. Second, the High-End Hardware Gap exists because while methods like Slicing Aided Hyper Inference (SAHI) prove that tiling improves small object detection by 20–30\% \cite{akyon2022sahi}, these are typically designed for expensive workstations. By implementing a Distributed Tiling architecture and using LBCNNs to shrink the model size, this project achieves high-definition accuracy previously reserved for expensive PCs. This approach ensures total data privacy by keeping processing local and provides a low-cost, scalable solution for industrial safety and smart energy management.

Smart building sensors are moving away from simple motion detection toward high-fidelity computer vision. This shift is necessary because traditional sensors, like Passive Infrared (PIR) and Radar, often fail to detect stationary people. In fact, research by Ahmad et al. \cite{Ahmad2021} shows that PIR sensors have a 0.0\% reliability when an occupant sits still. While PC-based models can reach an 85.0\% Mean Average Precision (mAP), they are too bulky for cheap hardware. To bridge this gap, the project uses INT8 quantization. This strategy shrinks the model’s memory footprint by four times while keeping 99.7\% accuracy \cite{nagel2021white}. By following the ESP32-S3 benchmarks \cite{esp32s3_datasheet}, we can fit these algorithms into a tiny 512 KB SRAM limit. Furthermore, Wu et al. \cite{wu2016qcnn} proved that focusing on "response fidelity" allows for 20x compression. This is a vital step to fix the slow 800 ms--1200 ms latency found in standard edge devices.

Efficiency also comes from how the AI is built. Iandola et al. \cite{iandola2016squeezenet} introduced SqueezeNet, which uses "Fire Modules" to cut parameters drastically. Taking this further, Juefei-Xu et al. \cite{juefei2017lbcnn} developed LBCNNs. By replacing standard filters with fixed binary kernels, they achieved a 169x parameter reduction. This project also tackles the "Information Loss Gap." Instead of shrinking a 4K image and making it blurry, we use data partitioning via tiling. Li et al. \cite{li2020pytorch} demonstrated that tiling preserves the tiny spatial details needed for accurate detection. Although the SAHI framework \cite{akyon2022sahi} can boost accuracy by 20--30\%, it usually requires a powerful PC. This project fills that gap by using the ESP-NOW protocol. As benchmarked by Jeong et al. \cite{jeong2021}, this protocol keeps communication latency as low as 1.115 ms. By combining LBCNN efficiency with this distributed tiling architecture, we create a low-cost, private alternative to expensive industrial monitoring systems.
